# FastAPI Project Structure Reference

Standard project structure for CT AI Lab web applications deployed to Azure App Service.

---

## Directory Layout

```
project_root/
├── app_web/
│   ├── main.py                 # FastAPI app, routes, API endpoints
│   ├── [business_logic].py     # Pure Python business logic (no framework deps)
│   ├── [data_file].json        # Static data (hospital data, lookup tables, etc.)
│   ├── [context_file].md       # AI context documents (RAG source material)
│   ├── templates/
│   │   └── index.html          # Jinja2 template (single-page app)
│   └── static/
│       ├── css/
│       │   └── ct_custom.css   # CT.gov design system styles
│       ├── js/
│       │   └── app.js          # Client-side logic (vanilla JS)
│       └── images/
│           └── ct-logo.svg     # Static assets
├── requirements.txt
├── startup.sh
├── .env.example
├── .gitignore
├── CLAUDE.md                   # Project conventions for Claude Code
└── ARCHITECTURE.md             # System documentation (generated by /explain)
```

## main.py Pattern

```python
import os
from pathlib import Path
from fastapi import FastAPI, Request
from fastapi.staticfiles import StaticFiles
from fastapi.templating import Jinja2Templates
from fastapi.responses import StreamingResponse, JSONResponse
from dotenv import load_dotenv

load_dotenv()

app = FastAPI()

BASE_DIR = Path(__file__).resolve().parent
app.mount("/static", StaticFiles(directory=BASE_DIR / "static"), name="static")
templates = Jinja2Templates(directory=BASE_DIR / "templates")


@app.get("/")
async def home(request: Request):
    """Render the main page, passing server-side data to the template."""
    return templates.TemplateResponse("index.html", {

        "translations": TRANSLATIONS,
        # ... other server-side data
    })
```

## Business Logic Module Pattern

Keep business logic in a separate file with NO web framework imports:

```python
# app_web/screener.py (or whatever the domain is)

TRANSLATIONS = {
    "en": { "key": "English text", ... },
    "es": { "key": "Spanish text", ... },
}

def calculate_something(input1, input2):
    """Pure function — easy to test, no request/response objects."""
    # business logic here
    return result
```

This separation means:
- Business logic is testable without spinning up a server
- main.py is thin — just routing and request/response handling
- Translations live with the business logic, not in the template

```javascript
function showStep(stepId) {
    ALL_STEPS.forEach(id => {
        document.getElementById(id).style.display = (id === stepId) ? '' : 'none';
    });
    window.scrollTo({ top: 0, behavior: 'smooth' });
    // Focus the step heading for screen readers
    const heading = document.getElementById(stepId)?.querySelector('h3[tabindex="-1"]');
    if (heading) setTimeout(() => heading.focus(), 100);
```

## Conventions

1. **Vanilla JS only** — no React, no npm, no build step
2. **Single HTML file** — step-based navigation, not multi-page
3. **All user text has `data-i18n`** — prepared for EN/ES from the start
4. **Business logic is framework-free** — testable without FastAPI
5. **CSS custom properties** — all colors defined in `:root`, no hardcoded hex in components
6. **Mobile-first responsive** — breakpoint at 700px
7. **Accessible by default** — skip link, aria attributes, focus management, keyboard nav
